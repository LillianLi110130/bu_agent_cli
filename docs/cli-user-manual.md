# CLI Agent 用户使用手册

## 这是什么
CLI Agent 是一个在终端中使用的智能助手。你可以把它理解为“会对话、会看项目、会按要求执行任务”的协作伙伴。它适合用来处理日常开发工作，比如理解代码、修改文件、生成文档、排查问题、做代码评审，以及按固定流程推进需求工作。

## 快速开始
启动 CLI 后，直接输入自然语言即可开始，例如：

```text
帮我梳理这个仓库的目录结构
修复这个报错，并补上测试
把 README 改成中文版本
```

如果不确定能做什么，先输入：

```text
/help
```

## 它现在能做什么

### 1. 日常对话式协作
最常见的用法就是直接提需求。你可以让它：

- 阅读并总结当前项目
- 修改代码或文档
- 解释某段逻辑
- 排查报错和回归问题
- 帮你写测试、写说明、写提交说明

一般来说，直接用自然语言描述目标就够了，不需要先想命令。

### 2. 技能增强模式
当你希望它进入某种“专门工作方式”时，可以用 `@技能名`。

常见例子：

```text
@brainstorming 帮我先发散几个实现方案
@systematic-debugging 帮我定位这个问题的根因
@test-driven-development 先写测试，再实现
@writing-plans 帮我把需求拆成执行计划
```

查看当前可用技能：

```text
/skills
```

## 3. 图片理解
CLI 支持把本地图片直接交给 agent 分析，适合看界面截图、报错截图、流程图等。

示例：

```text
@"D:\screens\bug.png" 请分析这张图里的报错
@"D:\design\page.png" 请指出这个页面的可用性问题
```

这类方式适合做界面评审、错误定位和需求沟通。

## 4. 模型切换
如果你希望更换当前使用的模型，可以直接用：

```text
/model
/model list
/model show
/model <preset>
```

适合在“更快响应”和“更强效果”之间切换。

## 5. 会话管理
CLI 是连续对话式的，适合围绕一个任务持续推进。常用命令有：

- `/history`：查看最近命令记录
- `/reset`：重置当前会话，适合换任务时使用
- `/clear`：清空屏幕显示
- `/pwd`：查看当前工作目录

如果你是第一次在某个项目里使用，也可以先运行：

```text
/init
```

它会帮助生成项目协作说明，让后续回答更贴合当前仓库。

## 6. 后台任务
有些任务会运行较久，可以在后台继续处理。你可以用下面的命令管理：

- `/tasks`：查看后台任务列表
- `/task <task_id>`：查看某个任务详情
- `/task_cancel <task_id>`：取消任务

这适合长时间的整理、生成或流程类任务。

## 7. 插件与流程命令
CLI 还支持插件扩展。先用下面命令查看当前插件：

```text
/plugins
/plugins list
/plugins show <plugin>
```

当前更偏向工作流型能力，比如：

- `review-kit`：快速做 review 或总结
- `frontend-workflow`：做前端需求、设计、任务拆解
- `ta-workflow`：做分阶段任务拆解

常见示例：

```text
/review-kit:review auth flow
/frontend-workflow:requirement my_spec
/frontend-workflow:design my_spec
/frontend-workflow:tasks my_spec
```

## 8. 智能体管理
CLI 内置了不同角色的智能体配置，例如写代码、做评审、做前端、做数据分析等。普通用户不一定每天都会用到，但可以通过下面命令查看：

```text
/agents
/agents list
/agents show <name>
```

这部分更适合需要了解或维护“可用角色”的用户。

## 9. 权限与目录
如果 agent 需要访问新的目录，或你想控制它的操作确认方式，可以使用：

```text
/allow <path>
/allowed
/approval on
/approval off
/approval status
```

日常理解上，只要记住两点：

- 不在允许范围内的目录，先授权
- 高风险操作是否需要你确认，可以用 `/approval` 控制

## 推荐使用方式

### 最省心的用法
先直接说目标，再按需要补充限制条件。例如：

```text
修复登录流程里的空指针问题，顺便补一个回归测试
```

### 需要“特定工作模式”时
先用 `/skills` 看技能，再用 `@技能名` 发起请求。

### 任务太长时
用 `/tasks` 跟踪进度，不要反复重发同一个需求。

### 会话跑偏时
直接 `/reset`，重新开始一个更清晰的任务。

## 一句话总结
把 CLI Agent 当成一个可持续协作的终端助手即可：平时直接对话，专项任务用 `@技能`，流程化工作用插件命令，长任务用 `/tasks` 管理，换任务就 `/reset`。
