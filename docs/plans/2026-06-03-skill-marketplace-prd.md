# 团队私有技能管理与技能广场 PRD

**日期**：2026-06-03  
**状态**：Draft  
**版本**：v0.2  
**适用范围**：团队/企业私有技能广场 MVP

## 0. 关键决策摘要

| 主题 | 决策 |
| :--- | :--- |
| 广场范围 | 只做团队/企业私有技能广场，不做公共广场 |
| 发布机制 | 技能版本必须经过管理员审核后才能上架 |
| 审核对象 | 审核该版本技能包的全部内容，以及该版本的广场资料 |
| 版本模型 | 广场展示资料纳入版本控制，历史版本可追溯 |
| 当前版本 | 只能下载和安装当前最新审核通过版本 |
| 历史版本 | 只允许查看元信息、文件树和 `SKILL.md`，不允许下载和安装 |
| 安装位置 | 默认安装到用户级技能目录 `~/.tg_agent/skills/` |
| 一键安装 | 由浏览器在用户授权目录后写入本地文件 |
| 更新策略 | 不自动更新、不主动提醒，由用户手动下载或更新 |
| Agent 安装 | Agent 可搜索技能，但必须展示候选并经用户确认后安装 |
| 可见范围 | 按团队、部门、用户组控制可见性 |
| 评分评论 | MVP 不做评分和评论，只记录下载量和安装量 |

## 1. 产品定位

### 1.1 背景

当前项目已经支持本地多来源技能加载、`/skills` 查看、`skill_manage` 创建和更新用户级技能。技能通常以 `SKILL.md` 为入口，并可包含 `references/`、`scripts/`、`assets/`、`templates/` 等辅助文件。

现有能力偏个人本地管理。团队内如果要复用技能，通常需要手动传文件、手动解压、手动判断版本，缺少审核、权限、版本追溯和统一发现入口。本期建设团队私有技能广场，把技能包管理为组织资产。

### 1.2 产品目标

本期要解决四件事：

1. 发布者可以上传符合 `SKILL` 规范的 zip 技能包，并提交管理员审核。
2. 管理员可以审核技能包全部内容，审核通过后技能在权限范围内上架。
3. 使用者可以在技能广场搜索、查看、下载 zip，或通过授权目录完成一键安装。
4. Agent 可以根据自然语言寻找候选技能，但安装前必须让用户确认。

本期不改变 Agent 运行时的本地技能加载机制。技能安装后仍写入用户级目录 `~/.tg_agent/skills/`，由现有发现逻辑加载。

### 1.3 非目标

MVP 不覆盖以下能力：

- 公共技能广场。
- 评分、评论、举报治理。
- 自动更新和新版本主动提醒。
- 历史版本下载或安装。
- 远程删除、远程禁用用户本地已安装技能。
- 脚本语义扫描、敏感命令扫描、沙箱执行。
- 按项目或具体用户配置技能可见范围。

## 3. 术语与规则

### 3.1 技能包

技能包是 zip 文件，包内必须包含合法的 `SKILL.md`。可选目录包括：

- `references/`
- `scripts/`
- `assets/`
- `templates/`

审核时要覆盖 zip 包内全部内容，不只审核 `SKILL.md`。

### 3.2 广场展示资料

广场展示资料由发布者在发布版本时单独填写，包括：

- 展示名称
- 描述
- 分类
- 标签
- 可见范围

这些资料属于版本内容。某个版本审核通过后，该版本的展示资料被冻结，历史版本页要按当时内容展示。

### 3.3 调用名

调用名来自技能包内 `SKILL.md` frontmatter 的 `name` 字段。安装到本地后，用户通过 `@调用名` 使用技能。

广场展示名称可以与调用名不同。详情页必须同时展示展示名称和调用名。

同一团队内，当前有效技能的调用名必须唯一。如果新版本修改调用名，也必须重新校验唯一性并经过审核。

### 3.4 版本号

版本号由发布者手动填写，采用 SemVer。系统必须校验新版本号相对该技能历史版本递增。

只有当前最新审核通过版本可下载、可安装。历史版本只用于查看和追溯。

## 5. 领域模型

### 5.1 模型关系

```text
----------+        +--------------+
|  Skill   | 1 ---> | SkillVersion |
+----------+        +--------------+
     |
     | current_version_id
     v
当前最新审核通过版本
```

技能身份只保存稳定字段。所有会随版本变化的业务资料都保存在技能版本中。

### 5.2 技能身份

| 字段 | 说明 |
| :--- | :--- |
| `skill_id` | 技能唯一 ID |
| `team_id` | 所属团队 |
| `creator_id` | 创建者 |
| `current_version_id` | 当前最新审核通过版本 ID |
| `status` | 技能整体状态 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

技能身份不保存展示名称、描述、分类、标签、可见范围等资料。

### 5.3 技能版本

| 字段 | 说明 |
| :--- | :--- |
| `version_id` | 版本唯一 ID |
| `skill_id` | 所属技能 |
| `display_name` | 广场展示名称 |
| `description` | 广场描述 |
| `category` | 分类 |
| `tags` | 标签 |
| `visibility_scope` | 可见范围 |
| `call_name` | `SKILL.md` 中的调用名 |
| `semver` | 版本号 |
| `change_log` | 变更说明 |
| `zip_object_key` | zip 包存储地址 |
| `zip_sha256` | zip 包哈希 |
| `file_tree` | 包内文件树 |
| `skill_md_content` | `SKILL.md` 内容快照 |
| `validation_result` | 基础校验结果 |
| `review_status` | 审核状态 |
| `reviewer_id` | 审核人 |
| `review_comment` | 审核意见 |
| `submitted_at` | 提交审核时间 |
| `approved_at` | 审核通过时间 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 5.4 状态机

技能状态：

| 状态 | 说明 |
| :--- | :--- |
| `draft` | 尚无审核通过版本 |
| `listed` | 已上架 |
| `unlisted` | 管理员下架 |
| `disabled` | 管理员禁用 |

版本状态：

| 状态 | 说明 |
| :--- | :--- |
| `draft` | 草稿 |
| `pending_review` | 待审核 |
| `approved` | 已通过 |
| `rejected` | 已驳回 |

版本流转：

```text
draft -> pending_review -> approved
                         -> rejected
```

规则：

1. 首个版本审核通过后，技能可进入 `listed`。
2. 已上架技能提交新版本时，旧版本继续作为当前版本可下载、可安装。
3. 新版本审核通过后，自动成为当前最新版本。
4. 管理员下架只影响广场展示、下载和安装，不处理用户本地文件。

## 6. 核心流程

### 6.1 发布首个版本

| 步骤 | 动作 | 结果 |
| :--- | :--- | :--- |
| 1 | 发布者创建技能 | 系统创建 `Skill` 和首个 `SkillVersion(draft)` |
| 2 | 发布者填写展示资料、版本号、变更说明 | 草稿保存版本化业务资料 |
| 3 | 发布者上传 zip | 系统保存包并准备校验 |
| 4 | 系统执行基础校验 | 生成校验结果、文件树、`SKILL.md` 快照 |
| 5 | 发布者提交审核 | 版本进入 `pending_review` |
| 6 | 管理员审核通过 | 版本进入 `approved`，技能进入 `listed` |
| 7 | 管理员审核驳回 | 版本进入 `rejected`，发布者可修改后重新提交 |

### 6.2 提交新版本

| 步骤 | 动作 | 结果 |
| :--- | :--- | :--- |
| 1 | 创建者或管理员创建新版本草稿 | 基于同一 `skill_id` 生成新版本 |
| 2 | 填写新版展示资料和变更说明 | 新资料只属于该版本 |
| 3 | 上传 zip 并校验 | 校验版本号递增和调用名唯一性 |
| 4 | 提交审核 | 新版本进入 `pending_review` |
| 5 | 审核期间 | 当前旧版本继续可见、可下载、可安装 |
| 6 | 审核通过 | 新版本成为当前最新版本 |

### 6.3 管理员审核

管理员审核页需要展示：

- 展示名称、描述、分类、标签、可见范围。
- 调用名、版本号、变更说明。
- zip 包基础信息和校验结果。
- 完整文件树。
- `SKILL.md` 内容。
- `references/`、`scripts/`、`assets/`、`templates/` 等包内文件。

审核结论：

- 通过：版本变为 `approved`，并成为当前最新版本。
- 驳回：版本变为 `rejected`，必须填写审核意见。

### 6.4 下载 zip

| 步骤 | 动作 | 结果 |
| :--- | :--- | :--- |
| 1 | 用户进入详情页 | 系统校验用户是否命中可见范围 |
| 2 | 用户点击下载 zip | 系统只返回当前最新审核通过版本 |
| 3 | 下载成功 | 服务端记录下载事件 |

### 6.5 浏览器一键安装

浏览器不能静默写入固定路径。一键安装依赖用户授权目录写入能力。

| 步骤 | 动作 | 结果 |
| :--- | :--- | :--- |
| 1 | 用户点击一键安装 | 前端检查浏览器是否已有目录授权 |
| 2 | 首次安装时选择目录 | 用户选择 `~/.tg_agent/skills/` |
| 3 | 前端下载当前最新 zip | 只下载当前最新审核通过版本 |
| 4 | 前端解压 zip | 读取 `SKILL.md` 并解析调用名 |
| 5 | 前端检查同名冲突 | 判断是否允许写入 |
| 6 | 无冲突或用户确认升级 | 写入技能目录 |
| 7 | 写入安装记录 | 用于后续识别同一广场技能升级 |
| 8 | 上报安装事件 | 用于热度排序和审计 |

不支持授权写入能力的浏览器，可以降级为下载 zip。

### 6.6 本地同名冲突

安装前必须检查本地目录中是否已存在同名调用名。

| 场景 | 处理 |
| :--- | :--- |
| 本地不存在同名技能 | 允许安装 |
| 本地存在同名技能，且安装记录显示为同一广场技能旧版本 | 用户确认后允许覆盖更新 |
| 本地存在同名技能，但没有安装记录 | 默认禁止覆盖 |
| 本地存在同名技能，且记录显示不是同一广场技能 | 默认禁止覆盖 |

### 6.7 Agent 自然语言安装

| 步骤 | 动作 | 结果 |
| :--- | :--- | :--- |
| 1 | 用户提出自然语言需求 | 比如“帮我找一个代码审查技能并安装” |
| 2 | Agent 调用广场搜索接口 | 返回候选技能 |
| 3 | Agent 展示候选信息 | 展示名称、描述、版本、发布者、调用名 |
| 4 | 用户确认 | 选择要安装的技能 |
| 5 | 进入安装流程 | 触发浏览器授权安装 |

Agent 不允许静默自动安装技能。

### 6.8 历史版本查看

历史版本用于追溯，不用于运行。

历史版本页展示：

- 展示名称、描述、分类、标签、可见范围。
- 调用名、版本号、变更说明。
- 审核状态、审核人、审核意见、审核时间。
- 文件树。
- `SKILL.md` 内容。

历史版本不提供 zip 下载和一键安装。

## 10. 校验与安全

### 10.1 基础校验

| 编号 | 校验项 |
| :--- | :--- |
| VAL-01 | zip 包大小符合限制 |
| VAL-02 | zip 内不存在路径穿越 |
| VAL-03 | zip 内不存在绝对路径 |
| VAL-04 | zip 内包含合法 `SKILL.md` |
| VAL-05 | `SKILL.md` frontmatter 可以解析调用名 |
| VAL-06 | SemVer 版本号相对历史版本递增 |
| VAL-07 | 同一团队内当前有效技能调用名唯一 |
| VAL-08 | 可以生成文件树和 `SKILL.md` 内容快照 |

### 10.2 审核安全边界

MVP 不自动判断脚本是否安全。系统只做结构和路径类校验，管理员通过审核页查看脚本和其他文件内容。

### 10.3 本地安装安全边界

浏览器只能在用户授权的目录中写入文件。系统不能要求标准浏览器静默写入 `~/.tg_agent/skills/`。

本地同名调用名默认禁止覆盖。只有安装记录能证明本地技能来自同一个广场技能的旧版本时，用户确认后才允许覆盖。

## 11. 数据与指标

MVP 需要记录以下事件：

| 事件 | 用途 |
| :--- | :--- |
| 下载事件 | 热度排序、审计 |
| 安装事件 | 热度排序、审计 |
| 提交审核事件 | 审核效率统计 |
| 审核通过事件 | 发布追踪 |
| 审核驳回事件 | 质量分析 |
| 下架事件 | 治理审计 |
| 禁用事件 | 治理审计 |

下载量和安装量作为搜索排序辅助因子。MVP 不做评分、评论和排名运营。

## 12. 验收标准

### 12.1 发布审核

1. 发布者可以创建技能，填写版本化广场资料，上传合法 zip 并提交审核。
2. 系统可以完成基础校验，并阻止不合法包提交审核。
3. 管理员可以查看待审核版本的完整包内容、校验结果和版本资料。
4. 管理员可以审核通过或驳回版本。
5. 审核通过版本可以成为当前最新版本，并在权限范围内展示。
6. 已有上架版本提交新版本审核时，旧版本继续可下载和安装。

### 12.2 使用安装

1. 使用者只能看到自己有权限访问的技能。
2. 使用者可以搜索、查看详情、下载当前最新 zip。
3. 使用者可以通过浏览器授权目录后一键安装到 `~/.tg_agent/skills/`。
4. 一键安装可以识别本地同名冲突，并阻止误覆盖用户自定义技能。
5. 同一个广场技能的旧版本允许用户手动确认后覆盖更新。

### 12.3 Agent 安装

1. Agent 可以根据自然语言搜索候选技能。
2. Agent 必须展示候选技能信息。
3. 用户确认后才允许进入安装流程。
4. Agent 不允许静默自动安装。

### 12.4 历史版本与治理

1. 历史版本可以查看冻结资料、文件树和 `SKILL.md`。
2. 历史版本不能下载，不能安装。
3. 管理员下架技能后，广场不再展示、下载和安装该技能。
4. 下架和禁用不处理用户本地已安装文件。
