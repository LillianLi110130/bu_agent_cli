# CMBPal 打包与启动流程文档

## 一、概述

CMBPal 是一个基于 Python + Node.js 的桌面应用程序，采用 **conda-pack 打包 + 便携式分发** 的方式，无需传统安装程序，解压即可使用。

---

## 二、打包方式

### 2.1 Python 环境打包

- 使用 **conda-pack** 工具将完整的 Python 环境打包
- 输出文件：`cmb-pal-env.zip`
- 解压后得到 `CMBPal-unpacked` 目录，包含：
  - `python.exe` 及完整的 Python 运行时
  - 所有已安装的 Python 依赖包
  - `Scripts/` 目录（包含 pip、conda-unpack 等工具）

### 2.2 Node.js 环境

- 预打包 **Node.js v22.18.0** (Windows x64)
- 目录：`node-v22.18.0-win-x64/`
- 包含 npm 及相关模块

### 2.3 依赖修复机制

由于 conda-pack 在 Windows 上存在 bug（Issue #154），会破坏 Python 字符串转义，项目采用以下方案：

1. 缓存受影响的包 wheels：`cache/conda_unpack_wheels/`
2. 在安装时重新安装受影响包（主要是 `huggingface_hub`）

---

## 三、安装流程

### 3.1 入口

双击 **`deploy.bat`** 文件开始安装。

### 3.2 执行脚本

`deploy.bat` 会调用 `win_deploy.ps1` PowerShell 脚本，执行以下步骤：

| 步骤 | 操作 | 说明 |
|:---:|------|------|
| 1 | 解压环境 | 将 `cmb-pal-env.zip` 解压到 `CMBPal-unpacked` |
| 2 | 定位 Python | 查找 `python.exe` 位置（允许一层嵌套目录） |
| 3 | 运行 conda-unpack | 重写环境中的路径前缀，使其指向当前位置 |
| 4 | 修复 conda-unpack bug | 重新安装受影响的包（huggingface_hub 等） |
| 5 | 验证修复 | 测试 `huggingface_hub` 导入是否正常 |
| 6 | 复制配置 | 将 `config/` 目录复制到 `~/.copaw/` |
| 7 | 生成启动器 | 创建 `cmbPal_launcher.bat` 启动脚本 |
| 8 | 创建快捷方式 | 在桌面生成 `cmbPal_launcher.lnk` 快捷方式 |

### 3.3 配置文件

- 源目录：`config/`
- 目标目录：`C:\Users\<用户名>\.copaw\`
- 包含文件：`security_check_config.json`

---

## 四、启动流程

### 4.1 启动方式

通过桌面快捷方式 **`cmbPal_launcher.lnk`** 启动，或直接运行 `cmbPal_launcher.bat`。

### 4.2 启动器脚本逻辑

`cmbPal_launcher.bat` 执行以下操作：

```batch
@echo off
cd /d "%~dp0"

REM 设置环境变量
set "NODE_PATH=%~dp0node-v22.18.0-win-x64"
set "PYTHON_PATH=%~dp0CMBPal-unpacked\Scripts;%~dp0CMBPal-unpacked"
set "PATH=%NODE_PATH%;%PYTHON_PATH%;%PATH%"

REM 应用配置
set "COPAW_CMB_SKILLS_HUB_BASE_URL=http://skills.paas.cmbchina.cn"
set "COPAW_LOG_LEVEL=info"

REM 配置 SSL 证书
set "CERT_TMP=%TEMP%\copaw_cert_%RANDOM%.txt"
"%~dp0python.exe" -u -c "import certifi; print(certifi.where())" > "%CERT_TMP%" 2>nul
set /p CERT_FILE=<"%CERT_TMP%"
del "%CERT_TMP%" 2>nul
if defined CERT_FILE (
  if exist "%CERT_FILE%" (
    set "SSL_CERT_FILE=%CERT_FILE%"
    set "REQUESTS_CA_BUNDLE=%CERT_FILE%"
    set "CURL_CA_BUNDLE=%CERT_FILE%"
  )
)

REM 端口配置（默认 8088）
set "DEFAULT_PORT=8088"
echo Input port (Press Enter to use default %DEFAULT_PORT%)
set /p PORT=
if "%PORT%"=="" set "PORT=%DEFAULT_PORT%"

REM 初始化（如需要）
if not exist "%USERPROFILE%\.copaw\workspaces\default\agent.json" (
  "%~dp0CMBPal-unpacked\python.exe" -u -m copaw init --defaults --accept-security
)

REM 启动应用
"%~dp0CMBPal-unpacked\python.exe" -u -m copaw app --port %PORT% --log-level %COPAW_LOG_LEVEL%
```

### 4.3 关键环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `NODE_PATH` | Node.js 路径 | `./node-v22.18.0-win-x64` |
| `PYTHON_PATH` | Python 路径 | `./CMBPal-unpacked/Scripts;./CMBPal-unpacked` |
| `COPAW_CMB_SKILLS_HUB_BASE_URL` | 技能中心地址 | `http://skills.paas.cmbchina.cn` |
| `COPAW_LOG_LEVEL` | 日志级别 | `info` |
| `SSL_CERT_FILE` | SSL 证书路径 | certifi 自动检测 |
| `DEFAULT_PORT` | 服务端口 | `8088` |

## 六、注意事项

1. **无需管理员权限**：安装过程不写入系统目录，仅操作用户目录
2. **便携式部署**：整个目录可复制到其他位置使用
3. **端口占用**：启动时需确保 8088 端口（或自定义端口）未被占用
4. **首次初始化**：首次启动会自动执行 `copaw init` 初始化工作区
5. **SSL 证书**：自动配置 certifi 证书，适用于企业内网环境

## 七、常见问题

### Q1: 安装失败怎么办？
- 检查是否已安装 conda-pack
- 检查 Node.js/npm 是否在 PATH 中
- 检查 NSIS 是否安装（用于后续打包）

### Q2: 启动失败怎么办？
- 确认 `CMBPal-unpacked/python.exe` 存在
- 检查端口是否被占用
- 查看日志级别是否正确设置

### Q3: 如何修改端口？
- 启动时按提示输入自定义端口
- 或直接修改 `cmbPal_launcher.bat` 中的 `DEFAULT_PORT`
